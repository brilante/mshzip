# 명령 추적 및 검증 스킬 (Phase0→CC)

## 역할
사용자 명령을 기록하고, 이전 명령과의 불일치를 검사. **마인드맵 이력 노드와 연동하여 충돌을 감지**.

## 동작

### 1단계: 이력 수집
1. 마인드맵 이력 노드(`CFLA6IJAND`)에서 최근 명령 5건 읽기
   - API: `/기획 R CFLA6IJAND`
   - 읽기 실패 시 → 로컬 로그(`.claude/logs/session-*.log`)에서 최근 이력 대체

### 2단계: 충돌 검사
2. 현재 명령과 이전 5건 비교 → 충돌 패턴 감지:

| 충돌 유형 | 패턴 | 예시 | 심각도 |
|-----------|------|------|--------|
| 대상 충돌 | 같은 파일/함수 반복 수정 | A 수정 → A 재수정 | ⚠️ 경고 |
| 방향 충돌 | 추가 후 삭제 요청 | 기능 추가 → 기능 삭제 | 🔴 알림 |
| 범위 충돌 | 작은 수정 후 전면 리팩토링 | 버그 수정 → 전체 리팩토링 | 🔴 알림 |
| 목표 전환 | 완전히 다른 목표로 변경 | 인증 구현 → UI 개선 | ⚠️ 경고 |

### 3단계: 결과 처리
3. 충돌 감지 시:
   - 사용자에게 알림 (충돌 유형 + 이전 명령 인용)
   - 계속 진행 여부 확인
4. 충돌 없음 시:
   - 정상 진행
5. 명령 수행 완료 후:
   - 이력 노드에 기록 (`/기획 W CFLA6IJAND`)

## 목적
- Phase0의 "지표 미정" 방지
- 목표 변경 추적
- 종료조건 명확화
- **마인드맵 이력 기반 세션 간 연속성 확보**
